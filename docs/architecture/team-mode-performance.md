## Ruled-Out Issues (not valid after code review)

The following 14 issues were listed in the first pass. After a second line-by-line code review, none constitute real problems:

| Original # | Issue                                                | Reason Ruled Out                                                                                                                              |
| ---------- | ---------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| Orig 2     | Wake skips queuing                                   | Design decision: the `activeWakes` lock window lasts only a few ms (wake start → message sent); messages are persisted in the mailbox, nothing is lost |
| Orig 5     | N agents each attach 3+ listeners                    | Listener count is accurate but each walks an independent path - no duplicate delivery                                                         |
| Orig 7     | setTimeout self-loop never stops                     | Confirmed: the loop stops naturally when the pending queue is empty - it is not infinite                                                      |
| Orig 8     | Agent cold-start skipCache                           | skipCache is called only once during session initialization; the wake path uses the normal cache                                              |
| Orig 9     | PetIdleTicker 50ms                                   | Intentional design: the working state needs to sense mouse movement to trigger wake; native API costs <0.01ms per call                        |
| Orig 10    | Double IPC emit per chunk                            | teamEventBus is a main-process-internal EventEmitter and is not routed to the renderer; both paths serve distinct purposes                    |
| Orig 11    | allConversationIds memo invalidation                 | Only happens on agent add/remove, which is not a high-frequency operation                                                                    |
| Orig 14    | SWR revalidateOnFocus                                | Global key deduplication - N components share a single request                                                                               |
| Orig 15    | ThoughtDisplay 1s timer                              | Reasonable UX design; once per second is not high-frequency                                                                                  |
| Orig 16    | handleResponseStream O(N) lookup                    | N = number of team agents (2–10); nanosecond-scale, no measurable impact                                                                     |
| Orig 17    | taskManager.list full-table query                    | Functionally required (agent prompts need full task context); filtered by team_id                                                             |
| Orig 18    | SqliteTeamRepository read-then-write                 | Necessary cost of JSON blob storage; two operations combined total <0.2ms                                                                    |
| Orig 19    | 10,000 messages loaded per agent                     | Conversations are short at this stage; impact is negligible                                                                                  |
| Orig 20    | finalizeTurn writes idle_notification every turn     | Protocol requirement; mutual exclusion prevents double-writes                                                                                 |
